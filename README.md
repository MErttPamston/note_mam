# Учёт занятий

Новое приложение для записи учеников на занятия — календарь, группы/цены,
учёт болезней и оплат. Старый проект (Nota) не затронут и остаётся отдельно.

## Что внутри

- `server.js` — простой Node.js сервер (без Express и внешних сервисов),
  данные хранятся в файле `data.json` рядом с сервером.
- `public/index.html` — всё приложение (интерфейс), один файл, без сборки.
- `data.json` создастся автоматически при первом запуске.

## Запуск локально

```
npm install   # (зависимостей нет, но команда безопасна)
node server.js
```
Откройте http://localhost:3000 — код входа по умолчанию: **6551** (роль «админ»).

## Деплой на Render

Так же, как раньше:
1. Залить папку в GitHub-репозиторий.
2. Создать Web Service на Render, Build command пустой, Start command `node server.js`.
3. **Обязательно подключить постоянный диск (Persistent Disk)** и примонтировать
   его в папку проекта — иначе `data.json` будет стираться при каждом деплое
   (как было с `users.json` в старом проекте).
4. Никаких переменных окружения не требуется — сервисы вроде Gist/Telegram убраны.

## Как это устроено

- **Группы и цены** — вкладка «Группы и цены»: название (напр. «Лепка»),
  обычная цена и цена при болезни ученика.
- **Ученики** — можно привязать к группе и задать повтор:
  «каждую неделю по вторникам» или «второй вторник месяца» (любая неделя от
  1-й до 5-й или «последняя»).
- **Календарь** — показывает дни с занятиями; при открытии дня видно, кто
  должен прийти (по повторам), можно отметить «пришёл / заболел / пропуск» —
  цена пересчитывается автоматически (болезнь → цена из «сick price»,
  пропуск → 0 ₽). Можно добавить разовое занятие с произвольным названием
  и разово добавить/убрать ученика на конкретный день, не трогая его обычный
  повтор.
- **Оплата** — журнал оплат наличными + баланс за месяц (начислено против
  оплачено) по каждому ученику.
- **Коды** — несколько кодов входа с ролями: «админ» (полный доступ) и
  «просмотр» (только смотреть календарь/списки, без редактирования).
  Код `6551` создан по умолчанию с ролью админ — его можно оставить, заменить
  или добавить к нему другие в вкладке «Коды».
